# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities by emailing the project maintainers
directly. Do not file a public issue for confirmed or suspected security
vulnerabilities.

We will acknowledge receipt within 48 hours and provide a timeline for
resolution. Security fixes are expedited and released as soon as a verified
patch is available.

## Scope

The following are in scope:

- Code execution or data disclosure through crafted input
- Dependency vulnerabilities with a CVSS score of 7.0 or higher
- Exposure of secrets, tokens, or CI credentials

The following are out of scope:

- Theoretical attacks without a demonstrated exploit
- Social engineering of project contributors
- Denial of service through normal tool use

## Supported Versions

Only the latest major release receives security patches.
